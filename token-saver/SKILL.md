# DeepSeek v4 Token 优化技能

专为 DeepSeek v4 设计的 token 节省方案。结合 DeepSeek 的计价特点（输出比输入贵、长上下文额外计费）进行针对性优化。

## DeepSeek v4 Token 计价特点
- 输出 token 单价较高 → 要求输出极简
- 长上下文有额外开销 → 超过 8K 后费用递增
- 中文编码每字约 2-3 token

## 节省方法

### 1. 输出格式约束（省 30-50%）
在提问前加上：请用最简洁的方式回答，不要客套话，不要重复问题。

### 2. 上下文控制
- 定期清理无关记忆（forget 工具）
- 每轮对话控制在 3-5 次交互内
- 超过 8K 上下文的对话考虑开新会话

### 3. 结构化提问
- ❌ "你能不能帮我分析一下这个……"
- ✅ "分析：直接说结论，100字以内"

### 4. DeepSeek v4 推荐参数
- temperature: 0.3（减少冗余输出）
- max_tokens: 按需设上限
- 复杂任务先用 0.3 跑初稿，再用 0.7 细化

## 使用方式
说"省点 token"或"精简模式"即可自动应用优化。
